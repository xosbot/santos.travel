IVA Cortex — Executive Summary
Executive Summary

Where We Are,
Where We're Going

santos.travel has built a trusted reputation delivering premium inbound Kerala and South India experiences. As demand scales, the current manual operations model creates invisible ceilings — in quote turnaround speed, human capacity, and revenue per inquiry. This proposal outlines a phased technology transformation to break through those ceilings without sacrificing the personal touch guests expect.

The Current Challenges

⏳ Slow Quote Turnaround

Manually assembling itineraries from scattered vendor rate sheets takes hours, sometimes days. Leads go cold while competitors respond faster.

📋 Disconnected Data

Vendor rates, guest details, and booking confirmations live in spreadsheets, WhatsApp, and email — with no single source of truth or audit trail.

💸 Inconsistent Margins

Manual pricing calculation introduces human error risk. A single misquoted rate can erode profit on an entire group booking without detection.

🧠 Human Bottleneck

Every inquiry, quote revision, and on-trip query routes through the same small team — limiting growth and creating burnout risk at peak season.

📄 Inconsistent Branding

PDF itineraries and quote documents are manually assembled, leading to inconsistent formatting, missed upsells, and a less premium presentation.

🌐 Missed Scale Opportunities

No B2B API for travel agent partners, no automated SEO content engine, and no yield management — leaving significant revenue untapped.

🤖 The Solution: IVA Cortex — A Hybrid Automation Engine

IVA Cortex is not a chatbot. It is a full-stack intelligent operations platform where AI handles language (reading messages, writing itineraries) and code handles math (pricing, margins, database queries). This strict separation ensures financial precision is never left to chance, while every guest interaction remains warm, eloquent, and on-brand. The result: your team handles strategy; the system handles scale.

IVA Cortex — The 5 Core Pillars
System Architecture

The 5 Core Pillars
of IVA Cortex

Every pillar is an independently deployable module. They communicate through a shared PostgreSQL database and typed API contracts — meaning each can be built, tested, and rolled back without destabilising the others.

1

🎯 AI Intake & Triage — The Smart Front Door

An NLP agent monitors the WhatsApp Business inbox. When a lead arrives — even in fragmented, informal language — the AI extracts structured parameters: number of travelers, travel dates, destinations, budget signals, and group type. This data is written directly to the PostgreSQL CRM, a quote is automatically triggered, and a priority score flags high-value leads for immediate human attention.

WhatsApp API · LLM · PostgreSQL CRM
2

⚙️ Deterministic Quoting Engine — The Financial Heart

A strict code-only module (no AI involvement in numbers) that queries vendor rate tables, applies seasonal multipliers, calculates per-person pricing, adds configured margins, and assembles a complete quote. The final structured data is then passed to an LLM which writes only the descriptive itinerary text — never touching the numbers. Every quote is versioned and auditable.

PostgreSQL · Node.js/Python · LLM Text Layer
3

📄 Automated PDF Storefront — Instant Branded Proposals

Once a quote is approved, the system passes the structured data and generated text into a premium branded HTML template and renders a pixel-perfect PDF itinerary — complete with santos.travel branding, day-wise breakdowns, inclusions, hotel details, and terms. This replaces hours of manual document assembly with an automated, consistent, 30-second pipeline.

Puppeteer · Handlebars Templates · Cloud Storage
4

🗺️ On-Trip Concierge & Dispatch — AI for Active Travelers

A RAG (Retrieval-Augmented Generation) WhatsApp bot serves guests during their active trip. It retrieves their confirmed itinerary, hotel contacts, and driver details to answer questions instantly. A Voice AI agent (Vapi.ai) handles outbound dispatch calls to vendors and drivers — reducing the operations team's communication load by an estimated 70%.

RAG · WhatsApp Bot · Voice AI (Vapi.ai)
5

🚀 Growth Flywheels — Revenue Beyond Direct Bookings

Three independent growth engines: (1) A B2B Whitelabel API allowing international travel agents to query, price, and book itineraries under their own brand. (2) An autonomous SEO content engine that generates, optimises, and publishes destination guides to drive organic traffic. (3) A yield-management scraper that monitors competitor pricing to inform dynamic rate adjustments.

REST API · Content Engine · Yield Scraper

IVA Cortex — Technical Architecture
Design Philosophy

⚖️ The Hybrid Automation Principle

The core design rule of IVA Cortex is non-negotiable: AI handles language. Code handles math. Never the reverse.

LLMs are extraordinary at reading informal human messages, writing eloquent itinerary prose, and understanding intent. They are unreliable for arithmetic. Therefore, every financial calculation — vendor rates, seasonal multipliers, per-pax pricing, margin application — is performed by deterministic backend code querying a structured database. Only after the numbers are finalized does an LLM receive a typed JSON payload to generate descriptive text around those numbers. This guarantees that no guest is ever quoted an incorrect price due to an AI hallucination.

IVA Cortex — Next Steps
Next Steps

How We Begin

Phase 1 begins immediately upon approval. The first deliverable — a complete, reviewed PostgreSQL schema — will be ready within the first week. Here is what happens in the immediate term:

  • Schema Design Workshop

    A collaborative session to map existing vendor rate structures, seasonal pricing rules, and client data fields into the PostgreSQL schema. All current spreadsheet logic will be captured and formalized.

  • VPS & PostgreSQL Environment Setup

    Provision the production server, configure PostgreSQL with role-based access, set up daily automated backups, and establish encrypted connection protocols before writing a single line of application code.

  • Vendor Data Onboarding

    Migrate existing vendor rates, hotel contracts, vehicle pricing, and seasonal rules into the new database. This includes building the admin interface for ongoing rate management by the operations team.

  • Phase 2 Kickoff — Quoting Engine

    With the database ready, the deterministic quoting engine is built and tested against real historical quote scenarios to validate pricing accuracy before any automation is customer-facing.

  • Stakeholder Review at Phase 2 Completion

    Before advancing to AI/communication layers, a full demo of the quoting engine and PDF output will be presented. This is the most critical milestone — sign-off here confirms the financial foundation is correct.

Risk & Governance
🔒

Data Safety First

All existing vendor and client data is migrated — never replaced — with full backups at each phase. No production data is touched without explicit approval.

🔄

Rollback Capability

Every database migration has a corresponding rollback script. Phases are independently deployable, so any module can be paused or reverted without affecting others.

👤

Human Remains in Control

The AI never acts on a booking or financial transaction autonomously. All actions that involve money or confirmation require human sign-off until explicitly configured otherwise.

📊

Measurable Milestones

Each phase has defined, measurable acceptance criteria. Payment and progress are tied to working demonstrations — not just code delivery.
